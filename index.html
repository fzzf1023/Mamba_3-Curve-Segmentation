Overall Architecture
The model consists of four stages: input enhancement, Progressive Mamba spatial encoding, Query instance decoding + pixel topology branch, and centerline-assisted crossing/boundary-aware post-processing. The design targets three core failure modes: thin-line breaks, crossing confusion, and grid interference.

Task Definition, Symbols, and Evaluation Targets

Input image \(I\in\mathbb{R}^{3\times H\times W}\), output instance set and final extracted curve union:

\[ \mathcal{Y}=\{(M_k,s_k)\}_{k=1}^{N},\qquad \widehat{C}=\bigvee_{k=1}^{N} M_k. \]
Symbol Notes: \(M_k\in\{0,1\}^{H\times W}\) is the mask of the \(k\)-th curve instance; \(s_k\in[0,1]\) is the confidence score; \(N\) is the number of retained instances in the image; \(\widehat{C}\) is the merged final extraction mask used for curve-level evaluation.

Errors to Reduce

  1. Gap: one curve is broken into multiple segments.
  2. Bridge: two curves are incorrectly connected.
  3. Crossing confusion: instance assignment errors at intersections.
  4. Grid false positive: grid lines are mistaken for curves.

Metrics

  1. Final extraction: curve IoU / Dice, curve precision / recall, curve clDice.
  2. Topology diagnostics: skeleton recall, curve connectivity, fragmentation, and endpoint error over the final extracted curve union.
  3. Crossing diagnostics: crossing IoU / precision / recall / F1.
  4. Auxiliary dense-head diagnostics: centerline IoU / Dice, centerline precision / recall.
  5. SOTA-only instance separation: mAP50/75/50:95, PQ, split rate, merge rate, instance-count error, and optional COCO AP, all computed after post-processing rather than on raw queries.

Input Enhancement Layer (RGB/HSV/Scharr)
\[ x_0=\mathrm{Concat}\!\left(I,\mathrm{HSV}(I),\mathrm{Scharr}(I)\right) \in\mathbb{R}^{9\times H\times W}. \]
Detailed Explanation:
1) \(I\in\mathbb{R}^{3\times H\times W}\) is the raw image; \(\mathrm{HSV}(I)\) decomposes color into hue/saturation/value to reduce confusion between curves with similar brightness but different colors; \(\mathrm{Scharr}(I)\) provides edge direction and magnitude through \(G_x,G_y,\|G\|\), with stronger local gradient fidelity than the earlier Sobel baseline.
2) This design directly targets two common errors: missed thin curves under low contrast, and poor separation of colored curves on gray-white backgrounds.
3) After concatenating the three streams at the input layer, the downstream encoder can form early geometric responses without relying only on deep semantics, producing more separable line-like features sooner.

Meaning of HSV Components

H improves color discrimination; S suppresses gray/white background interference; V preserves luminance structure.

Meaning of Scharr Components

\(G_x,G_y\) provide directional gradients; \(\|G\|\) adds complementary activation for weak edges.

Engineering Benefit

When curves are thin and backgrounds are cluttered, the early feature layers form more separable responses.

Progressive Mamba Encoder (Structure and Tensor Flow)

Default chart preset: encoder_dims=(64,128,224,384) | blocks=(2,2,3,2) | chunk_size=64; the legacy preset restores (64,128,256,512) | (2,2,4,2).

Each fusion block:

\[ x_n=\mathrm{PE}(\mathrm{GN}(x)), \quad b_i=\mathcal{B}_i(x_n), \quad z=\mathrm{Fuse}_{SE}(\{b_i\}), \]
\[ x' = x + \mathrm{DropPath}(\gamma_1 \odot z),\quad x'' = x' + \mathrm{DropPath}\!\left(\gamma_2 \odot \mathrm{MLP}(\mathrm{GN}(x'))\right). \]
Detailed Symbol Notes:
1) \(\mathcal{B}_i\) denotes the \(i\)-th directional/scale branch, used to extract complementary context.
2) \(\mathrm{PE}(\cdot)\) preserves 2D coordinate semantics before sequence modeling, preventing position ambiguity after feature reordering.
3) \(Fuse_{SE}\) first performs global compression and then generates branch weights, suppressing redundant branches while emphasizing the most effective directions for the current image.
4) \(\gamma_1,\gamma_2\) (LayerScale) are per-channel residual scaling parameters; they typically start near small values to limit residual perturbation early in training.
5) DropPath randomly removes part of the residual paths during training, reducing deep co-adaptation, mitigating overfitting, and improving generalization.
6) This module targets two issues: insufficient long-range dependency modeling for thin curves, and unstable deep training. The former is covered by multi-branch scanning, and the latter is stabilized by LayerScale and DropPath.

Snake Scan Branch (Default H/V, Legacy H/V/D45/D135)

Offset prediction and deformable sampling:

\[ \Delta_d = \tanh(g_d(x_n))\cdot \alpha,\qquad x_d=\mathrm{GridSample}\!\left(x_n,\ \mathcal{G}+\Pi_d(\Delta_d)\right). \]

Sequence scanning and restoration:

\[ \tilde{x}_d=\mathrm{reshape}_d(x_d),\quad y_d=\mathrm{Mamba}_d(\tilde{x}_d),\quad b_d=\mathrm{reshape}_d^{-1}(y_d). \]

Offset alignment loss:

\[ \mathcal{L}_{snake}= \frac{1}{|D|}\sum_{d\in D} \frac{\sum \mathrm{SmoothL1}(\hat{\Delta}_d,\ T_d)\odot V\odot G_d} {\sum V\odot G_d}, \]
\[ \hat{\Delta}_d=\frac{\Delta_d}{\max(|\Delta_d|)},\qquad G_d=\mathbf{1}\!\left[\max(|\Delta_d|)\gt 10^{-4}\right], \]
\[ T_h=\frac{d_y}{\sqrt{d_x^2+d_y^2}},\quad T_v=\frac{d_x}{\sqrt{d_x^2+d_y^2}},\quad T_{d45}=\frac{d_y-d_x}{\sqrt{2}\sqrt{d_x^2+d_y^2}},\quad T_{d135}=\frac{d_y+d_x}{\sqrt{2}\sqrt{d_x^2+d_y^2}}. \]
Detailed Explanation:
1) The implementation supports \(D=\{h,v,d45,d135\}\); the default chart preset activates \(h,v\), while the legacy preset also enables the two diagonals.
2) \(g_d(x_n)\) predicts direction-specific offsets; \(\tanh(\cdot)\cdot\alpha\) limits offset magnitude so that sampling points do not jump too far and cause feature mismatch.
3) \(\mathrm{GridSample}(x_n,\mathcal{G}+\Pi_d(\Delta_d))\) performs deformable sampling on continuous coordinates; \(\Pi_d\) projects offsets onto the valid geometric component of direction \(d\).
4) \(\mathrm{reshape}_d\) rearranges 2D features into a directional sequence, \(\mathrm{Mamba}_d\) models long-range dependency on that sequence, and the result is restored back to 2D space.
5) The implementation aligns normalized offsets \(\hat{\Delta}_d\) with signed targets \(T_d\) projected from GT tangents \(d_x,d_y\), and supervision is applied only on the valid centerline region \(V\).
6) \(G_d\) is an offset-significance gate: when branch offsets are near zero, it prevents noise gradients from being amplified. This directly matches the off_max>1e-4 logic in the code.
FPN, Grid Bias, and Pixel Topology Heads
\[ P_i=\mathrm{Conv}_{1\times1}(f_i),\quad P_i\leftarrow P_i+\mathrm{Up}(P_{i+1}), \]
\[ F_{fpn}=\mathrm{Fuse}\!\big[P_1,\mathrm{Up}(P_2),\mathrm{Up}(P_3),\mathrm{Up}(P_4)\big],\quad F= \mathrm{Up}(F_{fpn})+\mathrm{Proj}(stem_{1/2})+\lambda_g B_{grid}\;(\text{optional}). \]
Detailed Explanation:
1) \(B_{grid}\) is a background bias map estimated by the pixel head, mainly capturing periodic background responses such as grid lines and ticks.
2) \(\lambda_g\) is a learnable scaling factor that controls the injection strength of the background-suppression component.
3) The additive fusion \(F+\lambda_g B_{grid}\) is used to suppress background while preserving backbone semantic magnitude, avoiding the over-suppression of weak curves that can happen with multiplicative gating.
4) The five outputs of the pixel topology head, centerline/crossing/boundary/direction/grid, correspond to connectivity, crossings, instance separation, directional consistency, and background suppression. In the current code path they mainly provide auxiliary geometric supervision during training, while centerline_logits, crossing_logits, and boundary_logits additionally enter post-processing at inference.
5) The current implementation exposes stem skip and grid bias as real backbone switches; stem skip remains enabled by default, while grid bias is disabled by default and should only be enabled when grid labels exist.

Pixel Head Outputs

  • centerline_logits
  • crossing_logits
  • boundary_logits
  • direction_vectors
  • grid_logits (only when enabled)

Functional Roles

  • Centerline: primary supervision for connectivity
  • Crossing: crossing-region recognition
  • Boundary: stronger instance separation
  • Direction: geometric directional consistency
  • Grid: auxiliary background suppression
Query Decoder (Main Instance-Segmentation Branch)

Initial queries:

\[ Q_0=Q_{learn}+Q_{content},\quad Q_{content}=\mathrm{Pool}(F,\mathrm{AttnMap}(F)). \]

DN queries (training phase):

\[ Q_{dn}=\mathrm{MaskedAvgPool}(F,GT)+\epsilon,\ \epsilon\sim\mathcal{N}(0,\sigma^2),\quad Q=\big[Q_{dn};Q_0\big]. \]

Layer-wise update:

\[ \bar{M}_l=\mathrm{AlignTopK}(Q_l,M_l),\quad Q_l\leftarrow Q_l + G_l\odot \bar{M}_l, \]
\[ \hat{Q}_l=\mathrm{SA}(Q_l+P_l;\mathcal{B}_{rel}),\quad \tilde{Q}_l=\mathrm{CA}(\hat{Q}_l+P_l,M_l;\mathcal{B}_{guide}), \]
\[ r_l=\sigma(h_l(Q_l)),\quad Q_{l+1}=r_l\odot \tilde{Q}_l + (1-r_l)\odot Q_l. \]
\[ \hat{y}_q=\mathrm{Cls}(Q_q),\quad \hat{u}_q=\mathrm{Quality}(Q_q),\quad \hat{m}_q=\langle \mathrm{MaskEmbed}(Q_q),F\rangle. \]
Detailed Explanation:
1) \(Q_{learn}\) provides learnable instance slots for stable capacity, while \(Q_{content}\) comes from content pooling on the current image and gives sample-adaptive initialization.
2) DN queries inject noisy GT anchors during training to reduce early matching jitter and the risk of empty-query collapse.
3) \(\mathrm{AlignTopK}\) lets each query aggregate only the most relevant subset of memory tokens, reducing global attention noise from complex backgrounds.
4) \(\mathcal{B}_{rel}\) is a relative-position bias between queries that improves relation modeling for crossing and parallel instances; \(\mathcal{B}_{guide}\) is the current legend guidance bias, which blends Lab color bias and Scharr-based structural bias using per-legend modality weights and automatically becomes zero when legends are absent.
5) \(r_l\) is a routing gate that controls the mixing ratio between new information and historical state: large \(r_l\) relies more on new evidence, while small \(r_l\) is more conservative.
6) \(\hat{y}_q,\hat{u}_q,\hat{m}_q\) output class, quality, and mask respectively, jointly reducing the error mode where a candidate gets a high class score but a poor mask.

The current default chart preset is tuned for legend-aligned charts with at most roughly ten curve instances.

Legend-Guided Module (Optional Path)

When legend patches are available, the legend path activates the current A/LCAB/LSAB/LGB/C/E stack:

  1. A: LegendPatchEncoder builds legend query priors from disentangled color and shape branches, then fuses them with a modality gate.
  2. LCAB: computes Lab-space color similarity bias for decoder cross-attention.
  3. LSAB + LGB: computes Scharr-based structural correlation bias and blends it with LCAB using per-legend modality weights.
  4. C: performs bidirectional InfoNCE alignment between legend features and matched queries.
  5. E: uses an adaptive legend gate and falls back to learned queries when legends are absent.
\[ q_i' = g_i\cdot q_i^{legend} + (1-g_i)\cdot q_i^{learned},\quad g_i\in[0,1]. \]
Current Default Status: The chart preset keeps this path enabled because legend-to-curve correspondence is assumed to be informative. It becomes fully effective when legend_patches are supplied; if the stock dataset pipeline is used unchanged, the gate safely falls back to learned queries and the legend-contrastive term evaluates to zero.
Matching Strategy and Loss System (16+2 Capacity, Reduced Chart Default)

Hungarian Main Matching

\[ \mathcal{C}= \lambda_{cls}\mathcal{C}_{cls} +\lambda_{mask}\mathcal{C}_{mask} +\lambda_{dice}\mathcal{C}_{dice} +\mathbb{1}_{style}\lambda_{style}\mathcal{C}_{style}. \]
Detailed Explanation:
1) \(\mathcal{C}_{cls}\) measures category matching difficulty, where correct high-confidence classes incur lower cost.
2) \(\mathcal{C}_{mask}\) measures pixel-wise binary consistency and mainly penalizes local misses and false positives.
3) \(\mathcal{C}_{dice}\) measures overall overlap quality and mainly corrects bias caused by foreground-area imbalance.
4) When the style head is enabled and style labels exist, \(\mathcal{C}_{style}\) participates in matching; under the default configuration the style head is disabled, so this term is inactive.
5) Hungarian matching selects a one-to-one primary assignment with a joint cost so that supervision considers both semantic correctness and geometric fit.

Total Loss Groups

\[ \mathcal{L}_{total}= \mathcal{L}_{A}+\mathcal{L}_{B}+\mathcal{L}_{C}+\mathcal{L}_{D}+\mathcal{L}_{E} +\mathcal{L}_{optional}. \]
\[ \mathcal{L}_{scalar} =\sum_i w_i\left(e^{-s_i}\mathcal{L}_i+\frac{1}{2}s_i\right), \qquad s_i=\log \sigma_i^2. \]
Current Default Aggregation Rule: The chart preset disables uncertainty weighting by default and directly minimizes the manually weighted sum \(\sum_i w_i\mathcal{L}_i\). The heavier legacy preset re-enables uncertainty weighting for ablation or replication.

CAPE Constraint Illustration

\[ \mathcal{L}_{cape} =\underbrace{\left(1-\frac{\sum P\odot S_{gt}}{\sum S_{gt}}\right)}_{\text{gap}} +\underbrace{\frac{\sum P\odot B_{inst}}{\sum B_{inst}}}_{\text{bridge}}. \]
Detailed Explanation:
1) The gap term \(1-\frac{\sum P\odot S_{gt}}{\sum S_{gt}}\) emphasizes skeleton coverage and directly suppresses breakage probability.
2) In the current SOTA implementation, \(P\) is the predicted centerline probability \(\sigma(\texttt{centerline\_logits})\), not the final query mask union.
3) The bridge term \(\frac{\sum P\odot B_{inst}}{\sum B_{inst}}\) constrains activation across instance boundaries, reducing bridge errors where two curves are merged into one.
4) CAPE is essentially a joint constraint on connectivity and separability: the former keeps each curve continuous, while the latter prevents different curves from being incorrectly linked.
Training Strategy and Learning Rate Schedule (Expanded View)

Optimizer and Parameter Groups

  • Optimizer: AdamW
  • Standard parameter-group learning rate: \(\eta_0\)
  • Mamba parameter-group learning rate: \(0.5\eta_0\)
  • Mamba parameter-group weight decay: 0.0 (standard groups use weight_decay=1e-4)
  • Defaults: lr=2e-4 | weight_decay=1e-4 | grad_clip=1.0

EMA Update

\[ \theta_{ema}\leftarrow \mu\theta_{ema} + (1-\mu)\theta,\qquad \mu=0.999. \]
Detailed Explanation:
1) \(\theta\) is the current iteration parameter set, and \(\theta_{ema}\) is the historical exponential moving average.
2) \(\mu=0.999\) makes EMA behave like a long-window low-pass filter, suppressing parameter jitter caused by batch noise.
3) Validation and inference use EMA parameters, which typically produce smoother and more stable quality scores and mask outputs.

Learning Rate Schedule (Expanded)

\[ \eta_t = \eta_0\cdot\frac{t}{T_w},\qquad 0\le t \lt T_w. \]
\[ \eta_t = \max\!\left( \eta_{min}, \frac{\eta_0}{2}\left[1+\cos\!\left(\pi\cdot\frac{t-T_w}{T-T_w}\right)\right] \right), \qquad T_w\le t\le T. \]
Detailed Symbol Notes:
1) \(\eta_t\): learning rate at optimization step \(t\), updated by step rather than by epoch.
2) \(\eta_0\): base learning rate; the Mamba parameter group uses \(0.5\eta_0\) to reduce early oscillation in the sequence module.
3) \(T_w\): number of warmup steps, satisfying \(T_w=\texttt{warmup\_epochs}\times\texttt{steps\_per\_epoch}\); the default is warmup_epochs=5.
4) \(T\): total number of training steps; after warmup, cosine decay is used for smooth reduction and lower late-stage oscillation.
5) \(\eta_{min}\): lower bound of the learning rate; in the code the standard parameter groups use \(10^{-6}\), and the Mamba groups use \(0.5\times 10^{-6}\).

Progressive Introduction of Complex Losses

\[ w_k(e)=w_k^{*}\cdot\min\left(1,\frac{e}{E_{ramp}}\right),\qquad k\in\{\mathrm{boundary,cape,pcc,snake\_offset}\}. \]
Detailed Explanation:
1) \(w_k(e)\) is the weight of the \(k\)-th complex loss at epoch \(e\), and \(w_k^{*}\) is its target weight.
2) \(E_{ramp}=10\) means the first 10 epochs linearly ramp the weight up, after which the loss stays at full weight.
3) Only positive-weight items are actually ramped; under the chart preset this means boundary, CAPE, PCC, and snake-offset are introduced progressively.
4) This progressive strategy lets the model first learn the primary convergent tasks (classification and masks), and then gradually adds topology-oriented constraints to reduce instability caused by multi-loss competition.

Resolution-aware fragment filtering and thin-line recovery:

\[ A_{min}=\max\!\left(\texttt{min\_pixels},\left\lceil H\cdot W\cdot \texttt{min\_area\_ratio}\right\rceil\right). \]
\[ m_q^{refine}= m_q^{strong}\ \cup \Big(\mathbf{1}[s_q\ge \tau_{recover}] \odot \mathbf{1}[\sigma(\hat{m}_q)\ge \tau_{low}] \odot \mathbf{1}[\sigma(C_{center})\ge \tau_{center}]\Big). \]

crossing/boundary-aware rules:

\[ n_{ov}=|\Omega_{ov}(m_i,m_j)|,\qquad \overline{C}_{overlap}=\frac{1}{n_{ov}}\sum_{p\in\Omega_{ov}}C_{cross}(p), \quad \text{evaluated only when } n_{ov}\ge 5. \]
\[ \mathrm{Decision}(m_i,m_j)= \begin{cases} \text{keep both}, & \mathrm{IoU}(m_i,m_j)\le t_{nms},\\ \text{keep both}, & \mathrm{IoU}(m_i,m_j)\gt t_{nms}\ \land\ n_{ov}\ge 5\ \land\ \overline{C}_{overlap}\gt 0.4\ \land\ \mathrm{IoU}(m_i,m_j)\lt t_{override},\\ \text{suppress}, & \text{otherwise}. \end{cases} \]
Detailed Explanation:
1) \(\mathrm{IoU}(m_i,m_j)\) is the intersection-over-union of candidate masks, and \(t_{nms}\) is the regular suppression threshold (default 0.7).
2) \(n_{ov}\) is the number of pixels in the overlap region; the mean crossing confidence \(\overline{C}_{overlap}\) is computed only when \(n_{ov}\ge 5\), exactly matching crossing_min_overlap=5 in the code.
3) Before duplicate suppression, each query mask is refined with a lower mask threshold only when the query score is strong enough and centerline confidence supports the recovered pixels; this improves thin-line recall without globally lowering the mask threshold.
4) When IoU exceeds the base threshold and crossing evidence is insufficient, the pair is normally treated as duplicate candidates and suppressed; however, strong boundary evidence can still keep both masks when the overlap is more consistent with adjacent curves than with true duplicates.
5) When IoU exceeds the base threshold and crossing evidence is sufficient, both instances are kept if IoU is still below the relaxed threshold \(t_{override}\) (default 0.92). Boundary-supported pairs use their own override threshold \(t_{boundary}=0.9\).
6) This logic matches the current _refine_curve_mask, connected-component filtering, and _should_suppress path in the code: it preserves deduplication ability while improving recall in crossing and near-parallel scenarios.

Supplementary Terminology Notes (Unified Definitions):
1) "Connectivity" means continuous reachability along the curve direction within the same instance; "separability" means different instances remain distinguishable in crossing or adjacent regions.
2) "Geometric consistency" is mainly constrained jointly by direction/snake/CAPE: the direction field provides local orientation, snake constrains sampling offsets, and CAPE constrains global connectivity and separation.
3) "Instance discrimination" is jointly handled by query classification, the quality branch, relation-aware decoding, and optional PCC contrastive supervision, corresponding to category, reliability, and cross-instance separation.
4) "Background suppression" is jointly handled by the grid branch and post-processing thresholds: the former suppresses periodic background in feature space, and the latter removes low-quality candidates at the instance level.
5) Every module in the terminology table maps to a specific error pattern, which makes it easier to align method design with error analysis.
